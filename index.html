TES Access Card Application

台北歐洲學校 門禁卡申請與帳務、製卡系統自動化對接流程說明

系統整合規格 v1.5

申請與串接作業流程

1

Step 1 : 家長登入 AOS APP 發出申請

前端操作
開啟 AOS APP ➔ 點選 " Access Card Request " ICON ➔ 選擇 "申請" 頁簽 ➔ 選擇學生 ➔ 點選 "Apply"
2

Step 2 : 跳轉至 Saho 申請頁面填寫申請資料

API 傳遞與導向
AOS System 傳送 API 給 Saho ➔ Redirect 至 Saho 申請頁面 ➔ 選擇卡片種類 (Student / Parent / Family Card) 和異動情況 (新申請/補卡/遺失等) ➔ 填寫相關資料送出
🔗 API 參數 (Payload): StudentKey FamilyKey Parent Account (SSO Email)
3

Step 3 : Saho 比對並傳送收費名單給 AOS System

比對與分流邏輯

Saho 比對確認申請是否收費：

分支 A. 不需收費

直接進入製卡流程。

分支 B. 需收費

傳送 API 給 AOS System 開單，並於介面提醒家長回 AOS APP 繳費後才製卡。

🔗 API 參數 (Payload): StudentKey FamilyKey
4

Step 4 : AOS System 自動開單並提供訂單編號給 Saho

自動化帳務
每筆資料自動產生一張 Activities 訂單 ➔ 回傳 "訂單編號" 給 Saho
📝 自動訂單設定參數：
• 訂購部門：依 StudentKey 所屬 APP 群組對應通知部門 BPS / BSHS / FS / GS
• 收費類別：Smart Card
• Section：CA
|
• Work Center：CA_W
|
• 收費項目：Smart Card Fee
|
• 金額：200
🔗 API 參數 (Payload): OrderId
5

Step 5 : Saho 獲取訂單繳費狀態完成後續流程

IP 白名單與輪詢
AOS System 提供 API ➔ Saho 排程主動同步 (設定 IP 白名單) ➔ 更新 Saho 製卡排程狀態 ➔ 通知家長取卡和開卡
💰 PaymentStatus 狀態判斷：
A. 確實繳費並銷帳
Paid
B. 訂單勾選 "免繳費"
Paid
⚠️ 勾選時需強制備註原因 ➔ 儲存後金額自動歸0 ➔ 付款狀態更新為已付清
C. 家長未繳款
Unpaid
🔗 API 參數 (Payload): OrderId PaymentStatus
🔍 申請進度查詢：
開啟 AOS APP ➔ 點選 " Access Card Request " ICON ➔ 選 "進度查詢" 頁簽 ➔ 連結 Saho 查詢登入頁 (僅提供網址連結跳轉，不提供API串接)
6

Step 6 : AOS System 更新 Student Card 資訊

核心卡片狀態同步
AOS System 同步 PowerSchool API ➔ 調整 Student Card 設定為最新狀態
💳 卡片汰換更新設定：
1. 新增卡片 (Status = Active)

系統自動新增，開始日設為同步當天。
若偵測到有其他正在使用的卡，將同步停用該卡片（舊卡結束日設為同步前一天）。

2. 停用卡片 (Status = Inactive)

比對後自動停用，結束日設為同步當天。

🔗 API 參數 (Payload): Studentkey CardNumber CardStatus

核心 API 清單與參數摘要

API 名稱 / 觸發階段 方向 (Sender ➔ Receiver) 必要攜帶參數 (Payload Parameters) 業務邏輯說明
申請 API (Step 2) AOS System ➔ Saho StudentKey, FamilyKey, Parent Account (SSO Email) 帶入學生與家長資訊跳轉 Saho 填寫頁，免除重複登打輸入。
收費名單傳送 API (Step 3) Saho ➔ AOS System StudentKey, FamilyKey 比對後需要收費的個案，主動推送給 AOS 用於開立單據。
訂單回傳 API (Step 4) AOS System ➔ Saho OrderId AOS System 自動產生 Activities 訂單後，回傳訂單編號給 Saho 儲存。
訂單狀態同步 API (Step 5) Saho ➔ AOS System (主動同步) OrderId, PaymentStatus Saho 設定 IP 白名單並同步付款狀態。Paid 則排程製卡，Unpaid 則暫停。
學生卡 API (Step 6)
AOS System (主動同步) ➔ PowerSchool
Studentkey, CardNumber, CardStatus AOS System 與 PowerSchool API 進行同步，自動判斷有效新卡並停用舊卡。
